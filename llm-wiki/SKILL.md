# LLM Wiki

基于 [Karpathy 的 LLM Wiki 模式](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)，让 LLM 持续构建和维护结构化知识库。

**核心理念**：知识编译一次、持续更新，而非每次查询时重新推导。你负责喂资料和提问，LLM 负责所有整理工作。

## 适用场景

- **个人成长**：目标、健康、心理、自我提升，积累日志和文章笔记
- **深度研究**：数周/数月持续阅读论文、报告，构建带演进论点的综合 wiki
- **读书伴侣**：逐章摄入，构建角色、主题、情节线索页面及其关联
- **团队知识库**：由 Slack、会议记录、项目文档等喂入，LLM 负责维护更新
- **其他**：竞品分析、尽职调查、旅行规划、课程笔记、爱好研究……

任何需要**持续积累和结构化组织**知识的领域都适用。

## 三层架构

| 层 | 说明 | 谁维护 |
|---|------|--------|
| **raw/**  | 原始资料（文章、论文、数据），不可变 | 用户 |
| **wiki/** | 结构化 Markdown 知识库（摘要、实体、概念、分析） | LLM |
| **Schema** | 行为约定文件，定义 wiki 结构和工作流 | 用户与 LLM 共同演进 |

Schema 统一使用 `AGENTS.md`。

## 灵活性原则

原文强调**一切都是可选和模块化的**：

- 目录分类（entities/concepts/...）仅为建议，按领域裁剪
- 纯文本项目不需要 assets 目录
- 小规模 wiki（< 100 篇资料）用 index.md 定位足够，无需搜索引擎
- 输出格式不限于 Markdown，可以是表格、Marp 幻灯片、matplotlib 图表
- Schema 应随使用**持续演进**，发现新模式时更新约定

初始化时与用户讨论，裁剪出适合其领域的版本。

## 四种操作

### 1. Init（初始化）

1. 确认 wiki 主题和存放路径
2. 创建目录结构（参考 [schema-template.md](schema-template.md) 中的默认结构）
3. 从模板生成 Schema，根据用户主题调整
4. 初始化 `wiki/index.md` 和 `wiki/log.md`
5. `git init` 并做首次提交

### 2. Ingest（摄入）

用户添加新资料到 `raw/` 后：

1. 完整阅读原始资料
2. 与用户讨论关键发现
3. 创建摘要页，创建或更新关联的实体/概念页
4. 更新 index.md，追加 log.md
5. 一次摄入可能触及 10-15 个页面

建议逐篇摄入、保持参与；也可批量摄入减少监督。

### 3. Query（查询）

1. 读 index.md 定位 → 读相关页面 → 综合回答并引用来源
2. 回答形式灵活：Markdown 页面、对比表格、Marp 幻灯片、matplotlib 图表、canvas
3. **有长期价值的回答应归档为新 wiki 页面**，让探索也能沉淀

### 4. Lint（巡检）

检查并修复：
- 页面间矛盾、被新资料取代的旧内容
- 孤立页面（无入站链接）、缺少独立页面的高频概念
- 缺失的交叉引用

巡检后 LLM 应**主动建议**：值得深入的新问题、值得补充的新资料来源。

## 工具链

| 工具 | 用途 |
|------|------|
| **Obsidian** | 浏览 wiki、图谱视图查看关联 |
| **Obsidian Web Clipper** | 浏览器一键裁剪网页为 Markdown |
| **图片本地化** | Obsidian 设置附件目录为 `raw/assets/`，绑定快捷键下载附件 |
| **Marp** | Obsidian 插件，从 wiki 内容生成幻灯片 |
| **Dataview** | Obsidian 插件，查询 frontmatter 生成动态表格 |
| **Git** | 版本历史、回滚、协作 |
| **qmd**（可选） | wiki 规模大了之后的本地混合搜索（BM25 + 向量） |

## 详细约定

所有页面格式、文件命名、index/log 格式、操作细节见 [schema-template.md](schema-template.md)。
初始化时从该模板生成项目的 Schema 文件。
